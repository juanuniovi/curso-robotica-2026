1
Instalar GitHub Desktop
Solo la primera vez. Tardará 5 minutos.
  1. Ve a desktop.github.com y descarga la versión para tu sistema operativo.
  2. Instala el programa normalmente (siguiente, siguiente, finalizar).
  3. Ábrelo e inicia sesión con tu cuenta de GitHub. Si no tienes cuenta, créala gratis en github.com.
  4. El profesor te añadirá como miembro del repositorio ugv-matlab. Recibirás un email de invitación — acéptalo.
2
Clonar el repositorio (solo la primera vez)
Esto descarga una copia del repositorio en tu ordenador.
  1. En GitHub Desktop: File → Clone repository
  2. En la pestaña GitHub.com, busca ugv-matlab en la lista.
  3. Elige dónde guardarlo. Recomendado: dentro de tu carpeta de MATLAB (Documentos/MATLAB/ugv-matlab).
  4. Pulsa Clone. Espera a que descargue.

3
Fetch + Pull — descargar novedades
Hazlo siempre al empezar cada sesión de laboratorio. Descarga los modelos nuevos que el profesor haya publicado y los cambios de otros equipos.
  1. Abre GitHub Desktop. Verás el repositorio ugv-matlab.
  2. Pulsa el botón Fetch origin en la barra superior.
  3. Si aparece el botón Pull origin, púlsalo también.
  4. Cuando termine, tus archivos locales están actualizados.

¿Por qué es importante? Si no haces Pull antes de trabajar, puedes crear conflictos con el trabajo de tus compañeros. Es como guardar la versión antigua de un documento compartido encima de la nueva.
4
Commit — guardar tu trabajo con descripción
Hazlo al terminar cada sesión de trabajo. Un commit es una foto de tus archivos en ese momento, con un mensaje que explica qué hiciste.
  1. En GitHub Desktop, el panel izquierdo muestra los archivos que has modificado con una marca ✓.
  2. Revisa que solo están marcados archivos de tu carpeta de equipo. Si ves archivos de otros equipos o de base/, desmárcalos.
  3. En el campo Summary (abajo a la izquierda), escribe un mensaje breve y descriptivo.
  4. Pulsa Commit to main.

5
Push — subir al repositorio
Hazlo después de cada commit. Hasta que hagas Push, tus cambios solo existen en tu ordenador. El profesor y tus compañeros no pueden verlos.
  1. Después de hacer commit, aparece el botón Push origin en la barra superior.
  2. Púlsalo. En unos segundos, tus archivos están en GitHub.
  3. Puedes verificarlo: ve a github.com, busca el repositorio y comprueba que tus archivos aparecen.
Regla práctica: Fetch+Pull al llegar al laboratorio · Commit+Push al salir del laboratorio. Siempre, aunque no hayas terminado.

Nombres de archivo

Usa siempre este formato. Sin espacios, sin tildes, sin mayúsculas:

Formato: [subsistema]_s[semana]_[descripcion].[ext]
propulsion_s07_calibrado.slx
propulsion_s08_test_velocidad.slx

Problemas frecuentes

⚠ Problema
Aparece el mensaje Conflict al hacer Pull
✓ Solución
No intentes resolverlo solo. Avisa al profesor en ese momento — en 2 minutos se resuelve.
⚠ Problema
Veo archivos modificados de otros equipos en el panel de Changes
✓ Solución
Desmarca esos archivos antes de hacer Commit. Solo deja marcados los de tu carpeta.
⚠ Problema
El botón Push está desactivado o da error de permisos
✓ Solución
Cierra sesión en GitHub Desktop y vuelve a entrar. Si persiste, avisa al profesor.
⚠ Problema
Subí un archivo incorrecto o quiero deshacer el último commit
✓ Solución
No borres nada. Haz el cambio correcto y haz un nuevo commit con mensaje Corrijo [descripción].
⚠ Problema
El archivo .slx no se abre en MATLAB o da error de versión
✓ Solución
Comprueba que usas MATLAB R2023b o posterior. Los archivos .slx no son compatibles hacia atrás.
